# Contributing

Thank you for your interest in contributing to this project!

## Getting Started

1. Clone the repository
2. Run `node fetch-new-grain-meetings.js` to set up OAuth authentication
3. Make your changes
4. Test your changes
5. Submit a pull request

## Code Style

- Use consistent formatting (the project uses standard JavaScript style)
- Add comments for complex logic
- Keep functions focused and single-purpose
- Use descriptive variable and function names

## Testing

Before submitting a PR, please:
- Test that the script works with a fresh setup (no existing data)
- Verify that OAuth authentication works
- Test that meetings and transcripts are fetched correctly
- Ensure no sensitive data is logged or committed

## Pull Requests

- Keep PRs focused on a single feature or fix
- Include a clear description of what changed and why
- Update documentation if needed
- Ensure all user-specific data is properly gitignored

## Questions?

Open an issue for any questions or suggestions!
